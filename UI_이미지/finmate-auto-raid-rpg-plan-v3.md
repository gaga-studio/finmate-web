# FinMate · 마이데이터 기반 자동 레이드 금융 RPG 기획서 v3

> 추천 모험가의 검증된 금융 루틴을 내 상황에 맞는 목표로 바꾸고, 실제 금융행동의 변화를 3단계 자동 레이드로 보여주는 금융 습관 앱

---

# 0. 문서 목적과 결정 사항

이 문서는 v2의 아이디어를 보존하면서 제품 범위, 데이터 계산, 목표 생명주기, 개인정보와 AI 책임을 구현 가능한 수준으로 구체화한다.

## 0-1. v3의 핵심 결정

- 핵심 경험은 `추천 모험가 → 루틴 선택 → 내 목표로 가공 → 자동 레이드 → 퀘스트 → 기록`이다.
- 곰·물개·토끼·새의 네 동물은 유지한다.
- 소비·저축만 마이데이터 기반 정량 목표를 제공한다.
- 투자·금융지식은 MVP에서 점검·학습·기록 중심의 행동형 목표만 제공한다.
- 메이트 탭의 `친구 · 메이트 찾기 · 비교 탐색` 구조는 유지하되 MVP는 비교 탐색만 구현한다.
- 추천 모험가는 공개에 동의한 실제 익명 사용자다. 정확 금액과 식별 가능한 상품·종목은 보여주지 않는다.
- 자동 레이드는 짧은 전투 루프 뒤 현재 최고 지점에서 새 데이터 반영을 기다린다. 재도전 횟수는 만들지 않는다.
- 포인트·쿠폰·금융상품 가입 보상은 MVP에서 제외한다.
- AI는 목표값을 만들지 않고 코드가 만든 후보를 추천·설명한다.

## 0-2. 제품이 하지 않는 것

- 다른 사용자와 잔액·수익률·연봉을 경쟁시키지 않는다.
- 특정 상품 가입, 종목 매수, 투자금 증액을 퀘스트로 만들지 않는다.
- 앱 접속 시간이나 전투 반복만으로 금융상태와 목표 진행률을 올리지 않는다.
- 목표 미달, 일시정지, 데이터 지연에 손실이나 공개 순위를 부여하지 않는다.
- 한 익명 사용자의 특이한 금융값을 그대로 내 목표로 복사하지 않는다.

# 1. 서비스 정의

## 1-1. 제품 한 줄 정의

**FinMate는 나와 비슷한 실제 익명 모험가의 검증된 루틴을 내 금융상황에 맞는 목표로 바꾸고, 그 목표의 실제 진행을 네 동물과 자동 레이드로 보여주는 마이데이터 기반 금융 습관 앱이다.**

## 1-2. 핵심 사용자 가치

- 사회적 호기심 (나와 비슷한 사람이 실제로 유지한 습관을 발견)
- 낮은 해석 부담 (금융상태를 네 동물의 역할과 쉬운 리포트로 이해)
- 달성 가능한 시작 (정확 금액 복사가 아닌 내 기준의 세 목표 후보)
- 행동 연결 (목표와 직접 연결된 작은 퀘스트)
- 회고 가능성 (30일 금융 여정과 목표 변화 기록)

## 1-3. 게임의 역할

게임은 금융판단을 대신하지 않는다.

- 동물은 금융상태를 읽는 인터페이스다.
- 보스 HP는 코드가 계산한 목표 진행률의 시각 표현이다.
- 퀘스트는 실제 행동을 시작하게 돕는 단위다.
- 캐릭터 성장과 단계 해금은 행동을 지속시키는 비금전 보상이다.

# 2. 핵심 인과관계

## 2-1. 전체 흐름

```text
온보딩과 마이데이터 연결
→ 내 금융 기준선 진단
→ 목표 없는 홈 레이드 대기 상태
→ 메이트의 비교 탐색 진입
→ 추천 모험가 카드 선택
→ 성장 방식과 검증된 루틴 확인
→ 루틴 하나 선택
→ 코드가 가볍게·표준·도전 목표 후보 계산
→ 사용자가 목표 확정
→ 홈 3단계 자동 레이드 시작
→ 목표 연결 퀘스트 수행
→ 마이데이터·앱 행동 데이터 갱신
→ 코드가 스탯과 목표 진행률 재계산
→ 레이드·리포트·기록 갱신
```

## 2-2. 반드시 분리할 네 가지

| 개념 | 의미 | 변경 조건 |
| --- | --- | --- |
| 금융 스탯 | 현재 소비·저축·투자 판단 상태 | 마이데이터와 검증된 설정·점검 데이터 |
| 목표 진행률 | 확정한 한 목표까지 도달한 정도 | 목표 지표의 실제 데이터 변화 |
| 퀘스트 XP | 학습·기록·생활관리 행동의 누적 | 검증된 퀘스트 완료 |
| 레이드 연출 | 목표 진행률을 보여주는 시각 상태 | 코드가 계산한 목표·단계 상태 |

퀘스트 완료가 금융 스탯과 목표 진행률을 자동으로 올리지 않는다. 실제 지출·저축 변화가 확인되면 별도 계산으로 반영한다.

## 2-3. 이중 반영 방지 예시

`자동저축 설정 확인하기`를 완료한 경우:

1. 설정 확인 자체는 퀘스트 완료와 기록 배지에 반영한다.
2. 실제 저축 입금이 확인되면 물개의 저축 HP를 재계산한다.
3. 해당 입금이 활성 목표 지표에 포함될 때만 목표 진행률을 재계산한다.
4. 같은 행동을 XP, 금융 스탯, 보스 피해로 임의 중복 지급하지 않는다.

# 3. 온보딩과 전체 IA

## 3-1. 온보딩 순서

목표 후보 계산에는 금융 기준선이 필요하므로 마이데이터 연결 전에 목표값을 제안하지 않는다.

1. 생활맥락 (주거, 직업, 소득 규칙성, 부양 여부)
2. 돈 고민 진단 (소비, 저축, 투자, 금융지식 중 우선 고민)
3. 성향·위험도 (변화 속도, 투자 위험성향, 알림 선호)
4. 생활태그·공개설정 (자취, 월세 부담, 사회초년생, 익명 카드 공개 여부)
5. 마이데이터 연결 (수집 항목, 전송 주기, 철회 방법 확인)
6. 기준선 진단·캐릭터 생성 (네 동물의 초기 상태와 데이터 신뢰도 확인)

온보딩 완료 후 홈에는 목표 없는 전투장이 나타난다. `목표를 설정해주세요`를 누르면 메이트의 비교 탐색으로 이동한다.

## 3-2. 하단 내비게이션

```text
홈 · 메이트 · 퀘스트 · 기록
```

## 3-3. IA 트리

- 홈 (활성 목표와 자동 레이드를 확인)
  - 목표 미설정 (빈 전투장, 네 동물, 목표 설정 CTA)
  - 활성 레이드 (목표, 단계, 현재·최고 진행률, 마지막 동기화)
  - 새 데이터 반영 대기 (현재 최고 지점, 반영 대기 이유)
  - 동물별 리포트 (곰·물개·토끼·새 탭)
  - 단계 클리어 (다음 단계 해금, 기록 배지)
  - 목표 재설정 필요 (기준선 변화와 재확인 CTA)

- 메이트 (비교와 루틴 발견)
  - 친구 (Phase 2)
    - 친구·팔로잉 관리
    - 공개 동의된 루틴 피드
    - 친구와 함께한 기록
  - 메이트 찾기 (Phase 2)
    - 유사그룹 요약
    - 그룹 루틴 카드
    - 그룹 인사이트
  - 비교 탐색 (MVP)
    - 비교 조건 설정
    - 추천 모험가 카드
    - 익명 모험가 상세
    - 성장 방식 보기
    - 루틴 선택
    - 목표 후보 비교
    - 목표 확정

- 퀘스트 (목표를 돕는 작은 행동)
  - 오늘 요약
  - 목표 연결 퀘스트
  - 참여 가능
  - 진행 중
  - 완료·만료
  - 퀘스트 XP와 기록 배지

- 기록 (금융행동과 목표 이력)
  - 30일 금융 여정
  - 날짜 발판
  - 일일 기록 바텀시트
  - 목표·단계 이력
  - 예산안정도
  - 금융습관 회고

# 4. 네 동물과 금융 스탯

| 동물 | 영역 | 스탯 | 전투 역할 | 성장 근거 |
| --- | --- | --- | --- | --- |
| 곰 | 소비 | 소비 방어력 | 방어막·회피 | 예산 범위 유지, 반복지출 점검 |
| 물개 | 저축 | 저축 HP | 체력·회복 | 저축성 순유입, 비상금, 자동저축 유지 |
| 토끼 | 투자 | 투자 판단력 | 분석·약점 파악 | 위험성향 일치, 분산 점검, 투자 기초학습 |
| 새 | 퀘스트·금융지식 | 퀘스트 XP | 버프·정보 지원 | 퀴즈, 리포트, 기록, 회고 |

## 4-1. 공통 원칙

- 네 동물은 다른 사용자가 아니라 한 사용자의 금융영역을 나눠 보여준다.
- 낮은 스탯을 비만·부실·실패 같은 외형과 문구로 표현하지 않는다.
- 금융 스탯은 현금성 보상, 보스 클리어, 캐릭터 아이템으로 올리지 않는다.
- 목표와 같은 영역의 동물이 주 담당이 되고 나머지는 지원 연출만 한다.

## 4-2. 곰 · 소비 방어력

- 소비를 무조건 줄이는 지표가 아니다.
- 사용자가 정한 예산 범위 안에서 계획을 유지했는지 본다.
- 고정비와 재량지출을 분리한다.
- 극단적 절약을 추가 성과로 계산하지 않는다.

## 4-3. 물개 · 저축 HP

- 저축성 순유입, 자동저축 유지, 비상금 진행을 함께 본다.
- 예적금 만기 재예치는 새로운 저축으로 중복 계산하지 않는다.
- 중도해지는 원인 설명에 반영하되 벌점과 비난 문구를 사용하지 않는다.

## 4-4. 토끼 · 투자 판단력

- 투자금액, 수익률, 거래 횟수로 성장하지 않는다.
- 위험성향 확인, 자산 쏠림 점검, 분산 개념 학습을 반영한다.
- 특정 상품·종목과 매수·매도 행동을 추천하지 않는다.
- 투자금 입금·매매·수익 발생에는 XP·포인트·축하 효과를 지급하지 않는다.

## 4-5. 새 · 퀘스트 XP

- 소비·저축 생활관리, 금융퀴즈, 리포트 확인, 기록, 회고를 반영한다.
- 투자 거래나 투자금 증액은 퀘스트 XP 대상이 아니다.
- XP는 새의 외형, 설명 연출, 기록 배지를 바꾸지만 금융 스탯을 대신하지 않는다.

# 5. 금융데이터 계산 정책

## 5-1. 소득과 여윳돈

```text
여윳돈 = 기준 월소득 - 필수지출
```

- 정규소득: 최근 3개 완료월의 세후 소득 중앙값
- 불규칙소득: 최근 6개 완료월의 세후 소득 중앙값
- 필수지출: 월세, 관리비, 통신, 정기 교통, 보험, 부채상환
- 사용자가 잘못 분류된 소득과 필수지출을 수정할 수 있게 한다.

여윳돈이 0 이하이면 소비율·저축률·투자율을 만들지 않는다. `예산 기록하기`, `고정비 확인하기`, `지원정보 살펴보기` 같은 행동형 시작 목표만 제공한다.

## 5-2. 거래 분류

다음은 수입·지출·저축 성과에서 제외하거나 별도 이동으로 분류한다.

- 본인 계좌 간 내부이체
- 카드 승인 취소와 환불
- 대출 실행금과 원금 이동
- 예적금 만기 후 재예치
- 증권계좌와 연결계좌 사이 단순 자금 이동
- 중복 수집된 거래

분류가 불확실하면 자동 확정하지 않고 `확인 필요`로 표시한다.

## 5-3. 데이터 신뢰 상태

| 상태 | 조건 | UI 처리 |
| --- | --- | --- |
| 최신 | 템플릿 허용 기간 안에 동기화 | 정상 계산 |
| 반영 대기 | 새 행동 후 마이데이터 미도착 | 이전값 유지, 반영 대기 표시 |
| 오래됨 | 허용 기간보다 오래된 데이터 | 목표 진행 중지, 마지막 동기화 표시 |
| 부족 | 기준선 최소 기간 미충족 | 행동형 시작 목표만 제공 |
| 확인 필요 | 분류 충돌·중복·이상치 | 사용자 확인 전 목표값에 미반영 |

모든 홈·목표·리포트 화면에 마지막 동기화 시각과 데이터 상태를 표시한다.

## 5-4. 기준선 고정과 재설정

- 목표 확정 시 기준값, 데이터 기간, 분류 규칙, 계산 버전을 스냅샷으로 저장한다.
- 목표 도중 소득 또는 필수지출이 기준선 대비 20% 이상 변하면 목표값을 자동 수정하지 않는다.
- 상태를 `재설정 필요`로 바꾸고 새 기준선과 기존 기준선을 비교해 사용자에게 재확인을 요청한다.
- 목표값과 기준값이 같거나 분모가 0이 되면 후보를 생성하지 않고 유지형 또는 행동형 목표를 제안한다.

# 6. 추천 모험가 카드

## 6-1. 카드의 역할

추천 모험가 카드는 목표값을 직접 제공하는 카드가 아니다. 실제 사용자가 어떤 루틴을 유지했는지 보여주고, 사용자가 따라 할 행동을 선택하게 만드는 사회적 호기심의 진입점이다.

## 6-2. 출처 사용자 자격

- 익명 카드 공개에 명시적으로 동의
- 정량 루틴은 90일 이상, 행동 루틴은 30일 이상 검증 데이터 보유
- 데이터 최신성·완전성·분류 품질 통과
- 유사 출발점 그룹 안에서 루틴 지표가 P10~P90 범위
- 금융상품 가입·투자 거래·수익률을 성과로 삼지 않음
- 공개 철회, 데이터 삭제, 품질 저하 시 즉시 추천 제외

## 6-3. 유사 출발점 그룹

목표 영역별로 다음 조건을 사용한다.

- 소득 규칙성 (정규, 불규칙)
- 주거·고정비 부담 (부모 동거, 월세, 전세·자가 등 구간)
- 여윳돈 구간
- 시작 지표 구간 (초기 소비 범위, 저축률 등)
- 목표 영역 (소비, 저축, 투자 판단, 금융지식)
- 연령대·직업·생애단계 (사회적 유사성을 위한 보조 조건)

최소 표본은 30명이다. 표본이 부족하면 금융여력 조건은 유지하고 나이·직업·생활태그 범위만 순서대로 넓힌다. 그래도 30명 미만이면 실제 개인 카드처럼 보이지 않는 `그룹 루틴 카드`로 대체한다.

## 6-4. 카드 표시 정보

- 익명 닉네임과 모험가 외형
- 나와 비슷한 이유 2~3개
- 대표 루틴 1개
- 루틴 유지기간
- 성과의 비율·빈도 구간
- 데이터 기준일
- `금액 비공개 · 실제 루틴 · 공개 동의` 배지
- CTA `성장 방식 보기`

정확 소득, 잔액, 연봉, 거래 원문, 상품명, 종목명, 수익률은 표시하지 않는다. 불투명한 `매칭도 92%` 대신 구체적인 유사 이유를 보여준다.

## 6-5. 추천과 그룹 검증

추천은 코드가 수행한다.

1. 공개·데이터 품질·최소 기간을 하드 필터로 검사한다.
2. 소득 규칙성, 금융여력, 시작 상태, 목표 영역의 유사성을 계산한다.
3. 루틴의 유지기간과 데이터 최신성으로 정렬한다.
4. 출처 사용자의 실제 루틴 지표를 유사그룹 P25~P75와 템플릿 안전 범위로 제한해 목표 참고값을 만든다.
5. AI는 이미 선택된 카드의 추천 이유를 쉬운 문장으로만 설명한다.

## 6-6. 빌드 따라하기 흐름

```text
추천 모험가 카드
→ 성장 방식 보기
→ 루틴 근거·유지기간·공개 범위 확인
→ 따라 할 루틴 1개 선택
→ 내 기준선과 가공 원칙 확인
→ 가볍게·표준·도전 후보 비교
→ 사용자 최종 확정
→ 홈 레이드 시작
```

화면에는 `상대의 금액이 아니라 루틴과 유지 주기를 가져왔어요`를 명시한다.

# 7. 목표 템플릿과 개인화 엔진

## 7-1. 목표 관련 용어

| 개념 | 정의 |
| --- | --- |
| 루틴 | 추천 모험가가 실제로 반복한 금융행동 |
| 빌드 | 루틴, 유지기간, 검증 근거를 묶은 따라하기 단위 |
| 목표 템플릿 | 지표·기간·안전 범위·검증 방식이 사전 승인된 규칙 |
| 목표 후보 | 템플릿과 내 데이터로 계산한 가볍게·표준·도전 선택지 |
| 활성 목표 | 사용자가 후보 하나를 확인·동의한 뒤 진행 중인 목표 |
| 퀘스트 | 활성 목표를 돕는 작은 일일·주간 행동 |

## 7-2. `GoalTemplate`

MVP에서는 기존 미션 템플릿과 같은 버전 관리형 카탈로그로 운영한다.

| 필드 | 내용 |
| --- | --- |
| `templateId`, `version`, `active` | 식별, 계산 버전, 활성 상태 |
| `domain`, `goalType` | 소비·저축·투자 판단·금융지식과 목표 유형 |
| `metricKey`, `unit` | 측정 지표와 단위 |
| `baselinePolicy` | 기준선 기간과 제외 규칙 |
| `safeBounds` | 허용 최소·최대값 또는 적정 범위 |
| `minimumData` | 정량 목표 생성에 필요한 데이터 |
| `allowedDurations` | 허용된 목표 기간 |
| `verificationSource` | 마이데이터·설정·앱 활동 중 검증 출처 |
| `difficultyPolicy` | 가볍게·표준·도전 후보 계산 규칙 |
| `questTemplateIds` | 연결 가능한 퀘스트 |
| `bossTemplateIds` | 1·2·3단계 보스 |

## 7-3. MVP 목표 템플릿

| 템플릿 | 유형 | 기간 | 검증 | 금융 보상 |
| --- | --- | --- | --- | --- |
| 예산 범위 유지 | 범위 유지형 | 7일 | 지출·예산 기록 | 없음 |
| 구독 점검 | 행동형 | 7일 | 반복결제 확인 이벤트 | 없음 |
| 저축률 증가 | 증가형 | 30일 | 저축성 순유입 | 없음 |
| 비상금 증가 | 증가형 | 30일 | 지정 현금성 자산 순증가 | 없음 |
| 자동저축 확인 | 행동형 | 7일 | 설정 확인·입금 반영 상태 | 없음 |
| 투자 판단 점검 | 행동형 | 7일 | 위험성향·분산 개념 확인 | 없음 |
| 금융지식 확인 | 행동형 | 7일 | 퀴즈·리포트·용어 학습 | 없음 |

투자 판단 점검은 투자금액·거래·수익률을 목표로 만들지 않는다.

## 7-4. 목표 유형과 진행률

### 증가형

```text
진행률 = clamp((현재값 - 기준값) ÷ (목표값 - 기준값) × 100, 0, 100)
```

저축률과 비상금처럼 값이 커질수록 목표에 가까워지는 경우에만 사용한다.

### 감소형

```text
진행률 = clamp((기준값 - 현재값) ÷ (기준값 - 목표값) × 100, 0, 100)
```

불필요한 반복결제처럼 감소가 명확하게 좋은 지표에만 사용한다. 식비·생활비 전체에는 사용하지 않는다.

### 범위 유지형

```text
진행률 = clamp(목표 범위 안에 머문 검증 단위 수 ÷ 목표 검증 단위 수 × 100, 0, 100)
```

소비율과 예산 사용률에 사용한다. 범위보다 적게 썼다는 이유로 추가 성과를 주지 않는다.

### 행동형

```text
진행률 = clamp(검증된 완료 횟수 ÷ 목표 횟수 × 100, 0, 100)
```

예산 확인, 자동저축 설정, 투자 판단 점검, 금융학습에 사용한다.

## 7-5. 목표 후보 계산

추천 모험가의 루틴 지표는 다음 순서로 가공한다.

```text
출처 사용자의 실제 루틴 지표
→ 유사그룹 P25~P75 범위로 제한
→ GoalTemplate 안전 범위로 제한
→ 내 목표 참고값 확정
```

증가·감소형의 난이도 계수:

```text
가볍게 α = 0.25
표준   α = 0.50
도전   α = 0.75
```

```text
증가형 목표값 = clamp(기준값 + α × (참고값 - 기준값), 템플릿 안전 범위)
감소형 목표값 = clamp(기준값 - α × (기준값 - 참고값), 템플릿 안전 범위)
```

- 참고값이 내 상태보다 개선된 방향이 아니면 해당 후보를 만들지 않는다.
- 반올림 단위와 최소 변화량은 템플릿이 정의한다.
- 기준값과 목표값이 같아지면 정량 후보를 폐기한다.
- 범위 유지형과 행동형은 보간하지 않고 사전 승인된 횟수·기간 세트를 사용한다.
- 세 후보에는 목표값, 기간, 검증 방식, 마지막 기준 데이터와 예상 난이도를 함께 표시한다.

## 7-6. 목표 확정 예시

- 추천 모험가 루틴: `월급 다음 날 자동저축을 6개월 유지`
- 공개 카드: `저축률 15~20% 구간`
- 내부 그룹 검증: 출처 루틴이 유사그룹 P10~P90 안에 있음
- 내 기준선: `저축률 8%`
- 안전하게 가공한 참고값: `20%`
- 목표 후보: `가볍게 11% · 표준 14% · 도전 17%`
- 표준 선택 시: 여윳돈 `1,100,000원` 기준 30일 목표 `154,000원`
- 사용자 확인: 목표·기간·검증 출처·데이터 기준일·공개 범위 동의

AI는 `비슷한 출발점의 모험가가 유지한 먼저 저축 루틴을 내 여윳돈에 맞게 조정했어요`처럼 계산 결과를 설명한다.

## 7-7. 목표 생명주기

```text
CANDIDATE
→ CONFIRMED
→ ACTIVE
→ PAUSED
→ ACTIVE
→ COMPLETED | EXPIRED | CANCELLED
```

| 상태 | 의미 |
| --- | --- |
| CANDIDATE | 아직 사용자 동의를 받지 않은 후보 |
| CONFIRMED | 수치·기간·검증 방식·동의 버전 확인 완료 |
| ACTIVE | 데이터에 따라 진행률을 계산하는 유일한 활성 목표 |
| PAUSED | 진행·레이드·보상 계산을 중지한 상태 |
| COMPLETED | 목표 100%를 검증한 상태 |
| EXPIRED | 기간이 끝났으며 손실 없이 종료된 상태 |
| CANCELLED | 사용자가 취소한 상태 |

- 한 번에 ACTIVE 목표는 하나만 허용한다.
- 목표 확정 시 템플릿·계산·동의 버전과 기준선 스냅샷을 고정한다.
- PAUSED 중에는 진행률과 레이드를 갱신하지 않는다.
- 30일을 초과해 일시정지했거나 소득·필수지출이 20% 이상 변하면 재개 전에 새 후보를 계산해 재확인한다.
- 목표 템플릿이 개정돼도 진행 중 목표는 고정된 버전으로 끝낸다.
- 사용자가 취소하거나 공개를 철회해도 과거 개인 기록은 유지하되 다른 사용자 추천에서는 제거한다.

# 8. 홈과 3단계 자동 레이드

## 8-1. 레이드 상태

| 상태 | 화면 |
| --- | --- |
| NO_GOAL | 빈 전투장과 목표 설정 CTA |
| ACTIVE_ANIMATION | 새 데이터가 반영된 짧은 전투 연출 |
| WAITING_FOR_DATA | 현재 최고 지점에서 대기 |
| DATA_PENDING | 새 행동은 있으나 마이데이터 반영 전 |
| STAGE_CLEARED | 단계 보스 처치와 다음 단계 해금 |
| GOAL_COMPLETED | 3단계 완료와 목표 리포트 |
| PAUSED | 목표 일시정지 |
| NEEDS_RECALIBRATION | 기준선 변화로 재설정 필요 |

## 8-2. 3단계 구조

| 단계 | 전체 목표 진행률 | 클리어 |
| --- | --- | --- |
| 1단계 | 0~33% | 33% 이상 |
| 2단계 | 33~66% | 66% 이상 |
| 3단계 | 66~100% | 100% |

현재 단계 내부 진행률:

```text
1단계 = clamp(전체 진행률 ÷ 33 × 100, 0, 100)
2단계 = clamp((전체 진행률 - 33) ÷ 33 × 100, 0, 100)
3단계 = clamp((전체 진행률 - 66) ÷ 34 × 100, 0, 100)
```

보스 HP는 `100 - 현재 단계 내부 진행률`이다.

## 8-3. 전투 루프와 대기

1. 새 데이터가 반영되면 짧은 전투를 재생한다.
2. 캐릭터는 갱신된 진행률까지 보스 HP를 줄인다.
3. 단계 기준에 도달하지 못하면 전투 불능 연출을 반복하지 않는다.
4. 캐릭터는 현재 최고 지점에서 `새 데이터 반영 대기` 상태가 된다.
5. 사용자가 앱을 다시 열면 마지막 동기화, 새 데이터, 변경된 진행률, 다음 행동을 보여준다.

재도전 횟수, 오프라인 전투 횟수, 시간 누적 피해는 저장하거나 표시하지 않는다.

## 8-4. 진행률 하락과 단계 유지

- 실제 데이터에 따라 현재 진행률은 낮아질 수 있다.
- 현재 진행률과 최고 진행률을 함께 표시한다.
- 이미 해금한 단계는 다시 잠그지 않는다.
- 목표 완료 이후 상태가 달라져도 과거 달성 기록은 유지하고 현재 리포트와 분리한다.
- 하락을 실패·벌점·보스 회복으로 표현하지 않는다.

## 8-5. 보스 규칙

- 보스는 사람이나 일상 소비 자체가 아니라 사용자가 해결하려는 장애물을 상징한다.
- 예: `예산 흔들림`, `비상금 공백`, `정보 안개`, `용어 미궁`
- 사용자를 비난하는 `과소비 괴물`, `투자 겁쟁이` 같은 이름은 금지한다.
- AI는 승인된 보스 템플릿의 이름과 설명만 변형할 수 있다.

## 8-6. 단계 보상

- 캐릭터 외형의 작은 성장
- 다음 단계 보스 해금
- 목표·단계 기록 배지
- 새 설명 연출 또는 기록 꾸미기

현금, 포인트, 쿠폰, 금융상품 혜택, 투자 관련 보상을 지급하지 않는다.

투자 판단 목표는 단계 진행과 설명 리포트만 제공한다. 캐릭터 외형 성장, 보상 연출, 축하 효과는 제공하지 않는다.

# 9. 퀘스트

## 9-1. 퀘스트 목적

퀘스트는 보스 HP를 직접 줄이는 버튼이 아니다. 목표에 도움이 되는 행동을 시작하고 검증 상태를 보여준다.

## 9-2. 상태

- 참여 가능
- 진행 중
- 데이터 반영 대기

# 13. API 계약

현재 홈·비교·미션·기록 API는 유지하고 목표 생명주기용 계약을 추가한다.

## 13-1. 추천 모험가

### `GET /api/app/compare/adventurers`

- 입력: 목표 영역, 공개 가능한 생활맥락 필터, 커서
- 출력: `RecommendedAdventurerCard[]`
- 최소 표본·공개·데이터 품질을 통과한 카드만 반환

### `GET /api/app/compare/adventurers/{cardId}`

- 출력: 카드 상세, 성장 방식, 루틴 목록, 공개 근거
- 원본 사용자 ID와 정확 금융값은 반환하지 않음

## 13-2. 목표 후보와 확정

### `POST /api/app/goals/candidates`

- 입력: `cardId`, `routineId`
- 처리: 템플릿 연결, 기준선·참고값·안전 범위 계산
- 출력: LIGHT·STANDARD·CHALLENGE 후보, 계산 근거, 경고
- 데이터 부족 시 행동형 시작 목표만 반환

### `POST /api/app/goals/candidates/{candidateId}/confirm`

- 입력: 후보 ID, 동의 버전, 멱등성 키
- 처리: 후보 만료·데이터 변경·동의 상태 재검증
- 출력: 확정된 `ActiveGoal`
- 이미 활성 목표가 있으면 교체·일시정지 영향을 먼저 반환

## 13-3. 활성 목표

### `GET /api/app/goals/active`

- 현재 활성 목표, 레이드 상태, 데이터 상태, 마지막 동기화 반환

### `POST /api/app/goals/{goalId}/pause`

- 진행률 계산과 레이드 갱신을 중지

### `POST /api/app/goals/{goalId}/resume`

- 30일 초과·20% 기준선 변화·템플릿 문제를 재검증
- 문제가 있으면 재설정 후보를 반환하고 바로 활성화하지 않음

### `POST /api/app/goals/{goalId}/cancel`

- 목표를 CANCELLED로 변경하고 레이드 종료

### `GET /api/app/goals/{goalId}/history`

- 목표 상태와 진행률 스냅샷을 시간순으로 반환

## 13-4. 홈 응답 확장

`GET /api/app/home`에는 다음을 추가한다.

- `activeGoalSummary`
- `raidState`
- `currentProgress`, `highestProgress`
- `dataState`, `lastSyncedAt`
- `nextRecommendedAction`

확정·일시정지·취소 요청은 멱등성 키로 중복 처리를 막는다.

## 14-1. 코드가 담당하는 것

- 수입·필수지출·여윳돈과 금융지표 계산
- 공개·표본·이상치·데이터 품질 검증

- 목표값·범위·기간·진행률·단계 판정
- 퀘스트 검증과 데이터 상태
- 투자 관련 보상 차단

## 14-2. AI가 담당하는 것

- 코드가 선택한 추천 모험가의 추천 이유 설명
- 목표 후보 3개의 차이와 예상 난이도 설명
- 금융 리포트와 주간 회고의 쉬운 문장
- 승인된 보스·퀘스트 문구의 사용자 친화적 표현

AI는 코드가 전달한 ID와 구조화 데이터만 사용할 수 있다.

## 14-3. 구조화 출력

AI 목표 추천 출력:

```text
recommendedCandidateId
reasonCodes[]
summary
riskNotice
```

- 존재하지 않는 후보 ID를 반환하면 출력을 폐기한다.
- 금액·기간·진행률이 입력값과 달라지면 출력을 폐기한다.
- AI 실패 시 `표준 → 가볍게 → 도전` 기본 순서와 코드 근거만 제공한다.
- AI 설명 없이도 목표 선택·취소가 가능해야 한다.

## 14-4. 감사 로그

- 모델·프롬프트 버전
- 입력한 카드·후보 ID와 근거 코드
- AI 구조화 출력과 검증 결과
- 사용자에게 실제 노출한 문장
- 사용자 선택·수정·거절
- 오류 신고와 조치

금융 AI는 보조수단으로 사용하고 목표 확정은 사용자에게 남긴다. 거버넌스·합법성·보조수단성·신뢰성·금융안정성·신의성실·보안성 원칙을 운영 기준으로 둔다.

# 15. 개인정보와 게임화 안전

## 15-1. 카드 공개

- 별도 동의로 익명 카드 공개를 켜고 끌 수 있다.
- 공개 항목과 비공개 항목을 동의 화면에서 분리한다.
- 정확 금액 대신 구간, 빈도, 유지기간만 공개한다.
- 희귀한 태그 조합과 상품·종목·직장·지역 상세값은 억제한다.
- 최소 표본 30명과 위험도 검토를 통과해야 추천 인덱스에 들어간다.

## 15-2. 철회와 삭제

- 사용자가 공개를 철회하면 추천 인덱스와 새 카드 생성에서 즉시 제외한다.
- 캐시와 파생 추천은 다음 인덱스 갱신에서 제거하며 최대 24시간을 넘기지 않는다.
- 다른 사용자가 이미 확정한 목표는 원본 사용자를 식별할 연결을 제거하고 목표 템플릿·루틴 출처 유형만 남긴다.
- 법적 보존이 필요한 감사 기록 외에는 공개 원본값을 유지하지 않는다.

## 15-3. 게임화 가드레일

- 공개 순위와 사용자 간 기여도 경쟁을 만들지 않는다.
- 투자 거래, 투자금, 수익률에 XP·포인트·배지·축하 효과를 지급하지 않는다.
- 목표 미달과 데이터 지연에 손실·강등·보스 회복을 적용하지 않는다.
- 푸시는 데이터 반영, 사용자가 정한 일정, 개인정보 안내에만 제한한다.
- 시세 급등락, 실시간 종목, 반복 매매를 자극하는 알림은 만들지 않는다.
- 사용자가 레이드 연출을 줄이거나 끌 수 있게 한다.

## 15-4. 공식 기준

- 마이데이터는 정기 전송 주기가 1주~1개월일 수 있으므로 실시간 반영처럼 표현하지 않는다.
- 가명정보는 사전 준비, 위험성 검토, 가명처리, 적정성 검토, 안전한 관리 절차를 따른다.
- AI는 금융 소비자 이익과 보안을 우선하고 책임 주체와 인적 개입을 명확히 한다.
- 투자 게임화는 거래 빈도와 위험 감수를 높일 수 있으므로 교육·점검과 거래 행동을 분리한다.

# 16. MVP와 후속 단계

## 16-1. MVP에 포함

- 수정된 6단계 온보딩
- 곰·물개·토끼·새와 4개 리포트
- 목표 없는 홈과 목표 설정 CTA
- 비교 탐색의 실제 익명 추천 모험가 카드
- 성장 방식 보기와 루틴 선택
- 7개 목표 템플릿
- 가볍게·표준·도전 후보와 사용자 확정
- 목표 생명주기와 3단계 자동 레이드
- 전투 루프 후 새 데이터 반영 대기
- 목표 연결 퀘스트
- 30일 금융 여정과 일일 기록
- AI 설명과 감사 로그

## 16-2. MVP에서 제외

- 친구·팔로잉 기능
- 유사그룹 탐색 화면
- 실제 금융상품·투자종목 목록
- 투자금액·수익률 목표
- 공개 순위와 파티 기여도
- 포인트·쿠폰·현금성 혜택
- 확률형 보상 상자
- 자동 푸시 캠페인

## 16-3. Phase 2

- 친구의 공개 루틴과 함께한 기록
- 그룹 루틴 카드와 집계 인사이트
- 목표 템플릿 관리자 승인 시스템
- 실제 마이데이터 어댑터
- 알림 설정과 레이드 연출 강도 설정

## 16-4. Phase 3 검토

- 제휴 생활 혜택은 금융소비자 보호와 보상 유도 위험을 별도 검토한 뒤 도입 여부 결정
- 실제 상품 탐색은 교육 콘텐츠와 분리하고 비교·추천 규제를 충족하는 별도 흐름으로 설계

# 17. KPI와 사용자 검증

## 17-1. 핵심 퍼널

```text
추천 카드 노출
→ 성장 방식 보기
→ 루틴 선택
→ 목표 후보 확인
→ 목표 확정
→ 연결 퀘스트 시작
→ 검증된 첫 행동
→ 1단계 도달
```

## 17-2. 핵심 KPI

- 카드에서 성장 방식 보기 전환율
- 루틴 선택률
- 목표 후보 대비 확정률
- 검증된 첫 행동 완료율
- 4주 습관 유지율
- 1·2·3단계 도달률
- D7·D28 재방문율

## 17-3. 가드레일

- 목표 수정·거절률
- 목표가 과도하다고 느낀 비율
- 비교 압박·수치심 응답
- 데이터 분류·계산 불일치 신고율
- 공개 철회 처리 시간
- 투자 유도 문구·보상 발생 0건
- AI의 잘못된 후보 ID·수치 노출 0건

## 17-4. 초기 사용성 검증

금융 초보 청년 5~10명에게 다음 과제를 수행시킨다.

1. 목표 없는 홈에서 비교 탐색으로 이동
2. 추천 모험가가 실제 사람인지, 무엇이 비공개인지 설명
3. 성장 방식에서 따라 할 루틴 선택
4. 세 목표 후보의 차이 설명
5. 목표 확정 후 홈 레이드 확인
6. 연결 퀘스트 시작
7. 데이터 반영 대기 상태 설명

성공 기준:

- 전체 과제 완료율 80% 이상
- `상대의 돈이 아닌 루틴을 가져온다`는 이해 80% 이상
- 투자 판단력이 투자금·수익률 경쟁이 아니라는 이해 90% 이상
- 목표 적절성 중앙값 4/5 이상
- 개인정보 불안 요소를 화면별로 기록하고 개선

# 18. 검증 시나리오와 문서 QA

## 18-1. 계산·상태 시나리오

- 정상적인 저축률 증가 목표
- 여윳돈이 0 이하인 사용자
- 불규칙소득과 일회성 고액 입금
- 내부이체·환불·대출입금·만기 재예치
- 추천 그룹 표본 30명 미만
- 출처 루틴이 그룹 P10~P90 밖
- 오래된 마이데이터와 반영 대기
- 기준값과 목표값이 같은 후보
- 소득·필수지출 20% 이상 변화
- 후보 만료 후 확정 요청
- 목표 일시정지 30일 초과 후 재개
- AI가 잘못된 후보 ID·금액을 반환
- 추천 모험가의 공개 철회
- 목표 완료 후 현재 지표 하락

## 18-2. 개인정보·게임화 시나리오

- 태그 조합으로 원본 사용자를 추정할 수 없는지 확인
- 개인 카드 응답에 원본 ID·정확값·상품·종목이 없는지 확인
- 투자 행동에 XP·보상·축하 효과가 발생하지 않는지 확인
- 데이터 지연에 실패·보스 회복·손실이 없는지 확인
- 레이드 연출을 줄이거나 끌 수 있는지 확인

## 18-3. 문서 QA

- 온보딩 하위 항목 들여쓰기와 순서가 올바른지 확인
- `미션 XP`, `투자 공격력`, `재도전 횟수` 표현이 남아 있지 않은지 확인
- 쿠폰·상품·종목·수익률이 MVP 기능으로 남아 있지 않은지 확인
- 모든 목표가 템플릿·검증 데이터·사용자 확정에 연결되는지 확인
- 추천 모험가가 목표값을 직접 결정한다고 읽히는 문장을 제거
- 마지막 서비스 본질 문장을 원문 그대로 유지

# 19. 발표용 설명

## 19-1. 20초 설명

> 비교 탐색에서 나와 비슷한 실제 익명 모험가의 검증된 루틴을 발견합니다. 상대의 금액을 복사하지 않고 내 금융데이터로 가볍게·표준·도전 목표를 계산해 하나를 선택합니다. 이후 실제 행동이 데이터로 확인될 때만 네 동물과 3단계 자동 레이드가 앞으로 나아갑니다.

## 19-2. 핵심 차별점

- 실제 익명 사용자는 사회적 호기심과 루틴의 출처다.
- 목표 숫자는 내 데이터·검수된 템플릿·그룹 안전 범위로 계산한다.
- 투자금과 수익률이 아니라 투자 판단과 금융학습을 강화한다.
- 시간 누적과 반복 전투가 아니라 실제 데이터 변화가 레이드를 움직인다.
- AI는 목표를 만들지 않고 검증된 계산을 설명한다.

## 19-3. 서비스 한 줄 정리 · 추상화 본질

> 금융진입장벽을 느끼는 청년들에게 사회적 호기심과 동기를 유발하여 금융친화력을 길러준다.

- 사회적 호기심 (나와 비슷한 실제 모험가의 루틴을 발견)
- 동기 (내 목표, 작은 퀘스트, 자동 레이드)
- 금융친화력 (내 상태를 이해하고 행동하고 기록하며 스스로 판단하는 능력)
