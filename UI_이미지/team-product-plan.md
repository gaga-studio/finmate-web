> 공용 시작점이다. 화면 구현 세부 필드와 HTTP 계약은 OpenAPI, 상태 불변식은
> 도메인 문서가 우선한다.

| --- | --- | --- |
| 기획 | 제품 정의, 핵심 규칙, IA, 화면 명세 | 무엇을 왜 만들고 무엇을 제외하는가 |
| 디자인 | 전체 IA, 화면별 상세 명세, 공통 상태 | 각 화면의 주 질문과 한 가지 대표 행동은 무엇인가 |

기준에 매핑한다.

## 1. 제품 정의

### 서비스 한 줄

> 나와 비슷한 익명 모험가의 금융 루틴을 내 상황에 맞게 가져와, 실제 금융행동을 퀘스트와 레이드로 이어가는 금융 습관 앱

### 추상화 본질

> 금융진입장벽을 느끼는 청년들에게 사회적 호기심과 동기를 유발하여 금융친화력을 길러준다.

### 해결하려는 문제

- 금융 초보 청년은 정보가 없어서보다 `무엇부터 실행할지` 정하지 못해 행동을 미룬다.
- 평균·순위 중심 비교는 압박을 만들지만 구체적인 실천 방법을 알려주지 못한다.
- 금융교육, 자산 확인, 습관 기록이 분리되어 배운 내용을 반복 행동으로 연결하기 어렵다.

### 핵심 가치

- `사회적 호기심`: 나와 비슷한 익명 모험가가 유지한 루틴을 발견한다.
- `내 상황에 맞는 적용`: 상대 금액을 복사하지 않고 내 기준선으로 재계산한다.
- `작은 행동`: 중기 목표를 오늘 수행할 수 있는 퀘스트로 바꾼다.
- `검증된 성장`: 실제 또는 MVP의 합성 금융데이터가 확인된 뒤에만 금융 스탯과 레이드가 변한다.
- `이해 가능한 피드백`: 동물 캐릭터, 레이드, 일별 기록으로 변화 이유를 쉽게 읽는다.

### 대상 사용자

- 금융관리를 시작하고 싶지만 기준을 잡기 어려운 19~34세 청년
- 복잡한 용어보다 비슷한 사람의 구체적인 행동 예시가 필요한 사용자
- 수익률 경쟁보다 생활관리와 작은 성취를 원하는 금융 초보 사용자

### MVP 제품 경계

MVP는 `목표 확정 → 레이드 → 퀘스트 → 메이트 루틴 적용 → 기록`을 합성
금융데이터로 검증한다. 실제 마이데이터, 실제 금융상품 가입, 실제 투자·거래,
생성형 AI 호출은 제공하지 않는다.

## 2. 제품 핵심 개념과 인과관계

### 핵심 용어

| 용어 | 의미 |
| --- | --- |
| 기준선 | 합성 금융데이터로 계산한 현재 여윳돈·소비·저축·투자 판단 상태 |
| 주 목표 | 사용자가 이름·현재값·목표값·목표 월을 검토하고 확정한 중기 금융 결과 |
| 레이드 | 주 목표 진행률을 자동 전투로 보여주는 시각화 |
| 메이트 | 생활·금융 조건이 비슷한 익명 사용자 또는 그룹 |
| 루틴 | 메이트가 일정 기간 유지한 소비·저축·투자 판단 행동 |
| 빌드 따라하기 | 메이트 루틴을 내 기준선에 맞는 서브퀘스트로 가공해 적용하는 과정 |
| 퀘스트 | 목표와 루틴을 실행하기 위한 일일·주간 행동 |
| 기록 | 날짜별 금융활동·퀘스트·데이터 재계산 결과를 발판 여정으로 확인하는 영역 |

### 제품 인과관계

```text
퀘스트 행동 완료
└─ 퀘스트 XP·비현금성 내부 보상 지급

합성 금융데이터 동기화
└─ 금융 스탯 재계산
   └─ 목표 진행률 갱신
      └─ 레이드 최고 진행 지점 갱신
```

퀘스트를 눌렀다는 이유만으로 저축액, 소비율, 금융 스탯 또는 보스 체력이 임의로
변해서는 안 된다. 행동 완료와 금융데이터 검증은 다른 사건이다.

### 네 동물과 리포트

| 캐릭터 | 영역 | 화면에서 설명하는 것 | 금지 표현 |
| --- | --- | --- | --- |
| 곰 | 소비 방어력 | 예산 범위, 재량지출, 고정비 점검 | 체형 비하, 무조건 적게 쓸수록 좋다는 표현 |
| 물개 | 저축 HP | 저축 순유입, 저축률, 비상금 유지 | 단순 퀘스트 완료로 저축액 증가 |
| 토끼 | 투자 판단력 | 위험성향 일치, 분산 점검, 학습·리포트 확인 | 수익률 경쟁, 종목 추천, 매수 유도 |
| 새 | 퀘스트 XP | 행동 완료, 연속 실행, 금융학습 이력 | 금융 스탯과 같은 값으로 취급 |

### 목표가 없는 탐색 모드

- 온보딩 기준선 진단에서 `목표 설정` 또는 `일단 탐색하기`를 선택한다.
- `EXPLORE_ONLY`에서는 메이트의 친구·그룹·익명 모험가와 공개 루틴을 읽을 수 있다.
- 목표 확정 전에는 레이드 시작, 퀘스트 수락, 루틴 적용, 개인화된 하나 상품 정보가 잠긴다.
- 잠긴 행동을 선택하면 이유와 `목표 설정` 한 가지 복구 행동을 제공한다.
- 목표를 확정하면 상태가 `GOAL_ACTIVE`로 바뀌고 잠긴 기능이 열린다.

### 내비게이션 원칙

- 하단 탭은 `홈 · 메이트 · 퀘스트 · 기록` 네 개로 고정한다.
- 설정은 다섯 번째 탭으로 만들지 않고 홈 헤더에서 연다.
- 보스 탭은 현재 목표 맥락을 유지한 채 퀘스트 탭으로 이동한다.
- 기록은 장기 목표 지도가 아니라 최근 행동을 날짜별로 확인하는 발판 여정이다.
- 시연 시간 진행은 일반 사용자 IA와 분리한다.

## 4. 대표 사용자 흐름

### 목표를 확정한 핵심 흐름

```text
가입·온보딩
→ 합성 마이데이터 연결
→ 기준선 진단
→ 자금 모으기 선택
→ '유럽여행경비' 200만원→500만원, 2027년 1월 확정
→ 홈 레이드 시작
→ 네 동물 리포트 확인
→ 보스 탭
→ 퀘스트 탭의 빈 진행 중 상태
→ 추천 퀘스트 수락
→ 메이트 친구 현황
→ 메이트 찾기의 목표 달성 그룹
→ 익명 모험가 상세·비교 리포트
→ 월급날 먼저 저축 루틴 보기
→ 내 기준선에 맞춘 월 50만원 서브퀘스트 적용
→ 관련 하나 상품 정보 확인 후 복귀
→ 기록에서 7월부터 1월까지 자동저축 반영 확인
→ 홈 레이드 완료
```

### 목표를 미룬 흐름

```text
기준선 진단
→ 일단 탐색하기
→ 탐색 모드 홈
→ 메이트 친구·그룹·익명 모험가·공개 루틴 읽기
→ 루틴 적용 선택
→ 목표가 필요하다는 안내
→ 목표 설정·확정
→ 홈 레이드 시작
```

### 90초 결정적 시연

| 시간 | 장면 | 보여줄 내용 |
| ---: | --- | --- |
| 0–13초 | 온보딩 몽타주 | 생활맥락→돈 고민→금융성향→생활태그→합성 마이데이터→기준선 |
| 13–19초 | 목표 확정 | 자금 모으기, 유럽여행경비, 현재 200만원, 목표 500만원, 2027년 1월 |
| 19–29초 | 홈 | 레이드 시작과 곰·물개·토끼·새 리포트 전환 |
| 29–37초 | 퀘스트 | 보스 탭→진행 중 0개→`이번 달 저축 가능액 확인하기` 수락 |
| 37–51초 | 메이트 | 친구→메이트 찾기→비교 탐색→유럽여행 목표 달성 그룹·모험가 선택 |
| 51–64초 | 빌드 따라하기 | 모험가 리포트→월급날 먼저 저축→월 50만원 추천 서브퀘스트 적용 |
| 64–71초 | 하나 상품 | 검수된 상품 정보의 조건·기준일·유의사항만 확인하고 복귀 |
| 71–84초 | 기록 | 7월→1월 월 전환, 매월 50만원 자동저축 합성 데이터 반영 |
| 84–90초 | 홈 완료 | 여행자금 500만원, 보스 처치, 실제 데이터 반영 결과 요약 |

영상은 제품 API와 분리된 결정적 합성 fixture를 사용한다. `시간이 흘러서`가 아니라
각 월의 합성 저축 데이터가 동기화되었기 때문에 목표가 진행된다는 점을 보여준다.

## 5. 화면별 상세 명세

### 화면 상태 표기

- `MVP`: 실제 핵심 흐름으로 구현한다.
- `MVP·합성 읽기`: 합성 fixture를 읽어 탐색하지만 쓰기 기능은 제한한다.
- `시연 전용`: 발표 환경에서만 동작하며 운영 화면에는 노출하지 않는다.
- `후속`: IA에는 맥락만 남기고 첫 버전에서는 구현하지 않는다.

### 회원·온보딩

#### AUTH-01 · 회원가입·로그인 (`MVP`)

- **목적:** 이메일과 비밀번호로 사용자를 식별하고 중단한 온보딩 상태를 복원한다.
- **진입 조건:** 비로그인 사용자다.
- **필수 구성:** 이메일, 비밀번호, 회원가입/로그인 전환, 입력 오류, 이용 동의 안내.
- **대표 행동:** 가입 또는 로그인.
- **상태:** 입력 검증, 인증 실패, 네트워크 오류, 처리 중, 기존 세션 복원.
- **다음 화면:** 미완료 사용자는 마지막 온보딩 단계, 완료 사용자는 홈.

#### ONB-01 · 생활맥락 (`MVP`)

- **목적:** 금융여력을 비교하는 데 필요한 생활 배경을 파악한다.
- **필수 구성:** 소득 규칙성, 주거 형태, 고정비 부담을 중립적인 선택지로 제공한다.
- **대표 행동:** 현재 상황 선택 후 다음.
- **주의:** 직장명·주소·정확 소득을 익명 프로필에 공개한다는 인상을 주지 않는다.
- **다음 화면:** ONB-02 돈 고민.

#### ONB-02 · 돈 고민 진단 (`MVP`)

- **목적:** 사용자가 가장 먼저 개선하고 싶은 영역을 이해한다.
- **필수 구성:** 소비관리, 저축, 비상금, 투자 판단, 아직 모르겠음.
- **대표 행동:** 현재 고민 선택.
- **주의:** 이 선택만으로 목표나 금융상품을 자동 확정하지 않는다.
- **다음 화면:** ONB-03 금융성향.

#### ONB-03 · 금융성향 (`MVP`)

- **목적:** 소비 계획성, 투자 위험성향, 학습 선호를 확인한다.
- **필수 구성:** 짧은 상황 질문, 응답 요약, 수정 가능 안내.
- **대표 행동:** 응답 검토 후 확정.
- **주의:** 위험성향 결과는 종목 추천이나 투자금 증액에 사용하지 않는다.
- **다음 화면:** ONB-04 생활태그.

#### ONB-04 · 생활태그·익명 공개 설정 (`MVP`)

- **목적:** 유사그룹 탐색에 사용할 태그와 공개 범위를 사용자가 통제하게 한다.
- **필수 구성:** 자취, 사회초년생, 대학생 등 태그와 익명 카드 공개 항목.
- **대표 행동:** 태그와 공개 범위 저장.
- **상태:** 필수/선택 항목 구분, 공개 미동의 상태.
- **다음 화면:** ONB-05 합성 마이데이터.

#### ONB-05 · 합성 마이데이터 연결 (`MVP`)

- **목적:** MVP 금융 계산에 사용할 합성 데이터 어댑터를 연결한다.
- **필수 구성:** 동의 범위, 연결 데이터 범주, 연결 상태, 마지막 동기화 시각.
- **대표 행동:** 연결 동의 후 기준선 계산.
- **상태:** 연결 중, 실패, 데이터 부족, 연결 완료.
- **주의:** 실제 금융기관 연결인 것처럼 오해시키지 않고 `데모 데이터`를 명시한다.
- **다음 화면:** ONB-06 기준선 진단.

#### ONB-06 · 금융 기준선 진단 (`MVP`)

- **필수 구성:** 여윳돈, 소비율, 저축률, 투자 판단 행동, 데이터 상태, 계산 기준 설명.
- **대표 행동:** `목표 설정` 또는 `일단 탐색하기`.
- **상태:** `FRESH`, `PENDING`, `STALE`, `INSUFFICIENT`.
- **다음 화면:** 목표 설정은 GOAL-01, 탐색은 HOME-01.

#### GOAL-01 · 목표 선택·확정 (`MVP`)

- **목적:** 사용자가 레이드가 표현할 주 목표를 직접 검토하고 확정한다.
- **필수 구성:** 목표 유형, 사용자 지정 이름, 현재값, 목표값, 목표 월, 계산 근거.
- **대표 행동:** 목표 확정.
- **시연값:** 자금 모으기→유럽여행경비, 200만원→500만원, 2027년 1월.
- **상태:** 잘못된 범위, 기간 부족, 데이터 부족, 이미 활성 목표 존재.
- **주의:** AI나 메이트가 주 목표를 임의로 바꾸지 않는다.
- **다음 화면:** HOME-02 활성 레이드.

- **필수 구성:** 총자산 요약, 주 목표, 현재/최고 진행률, 보스, 네 동물, 활성 루틴,

#### PRODUCT-01 · 관련 하나 상품 정보 (`MVP·검수 카탈로그`)

- **금지:** 앱 내 가입, 자동 신청, 메이트 사용 상품 복제, 상품 열람 보상.
- **영향 없음:** XP, 금융 스탯, 목표값, 레이드 진행률을 변경하지 않는다.
- **다음 화면:** RECORD-01 또는 QUEST-01.

### 기록

#### RECORD-01 · 일별 대형 발판 여정 (`MVP`)

- **목적:** 한 달의 대표 금융활동과 데이터 반영 흐름을 날짜 순서로 탐색한다.
- **필수 구성:** 월 이동, 기록 일수, 월간 수입·지출·저축 요약, 1일부터 말일까지의
  큰 발판, 현재 날짜 캐릭터, 스크롤 힌트.
- **발판 규칙:** 날짜, 절대금액 기준 대표 금융활동 1개, 보조 활동 최대 2개,
  나머지 `+N`, 상세 진입 화살표만 표시한다.
- **대표 행동:** 날짜 발판 선택.
- **상태:** 완료 과거일, 오늘, 예정 미래일, 기록 없음, 데이터 반영 대기.
- **주의:** 날짜 순서가 건너뛰거나 발판에 모든 거래를 압축하지 않는다.
- **다음 화면:** RECORD-02 바텀시트.

#### RECORD-02 · 일일 기록 바텀시트 (`MVP`)

- **목적:** 선택한 날의 전체 활동과 계산 반영 결과를 발판 맥락 안에서 확인한다.
- **필수 구성:** 날짜, 기록 상태, 활동 내역, 지출 카테고리, 예산 사용·잔여,
  퀘스트·XP, 금융데이터 재계산 결과.
- **대표 행동:** 닫고 여정으로 복귀.
- **정보 위계:** 활동 목록을 첫 콘텐츠로 두며 같은 금액의 별도 요약 카드를 반복하지 않는다.
- **주의:** 바텀시트 뒤에 선택 발판 라벨을 중복 표시하거나 불필요한 하단 버튼을 넣지 않는다.
- **다음 화면:** RECORD-01.

#### DEMO-01 · 7월→1월 시간 진행 (`시연 전용`)

- **목적:** 6개월의 합성 자동저축 반영을 90초 영상 안에서 결정적으로 재생한다.
- **필수 구성:** 2026년 7월 시작 200만원, 8·9·10·11·12월과 2027년 1월에
  각 50만원 저축, 마지막 500만원, 각 월 동기화 완료 상태.
- **대표 행동:** 시연 컨트롤이 다음 fixture 월로 이동한다.
- **금지:** 운영환경 노출, 시간만으로 진행, 6회 이외의 임의 저축 이벤트.
- **다음 화면:** HOME-03 레이드 완료.

## 6. 공통 화면 상태

정상 데이터 화면만으로는 완료로 인정하지 않는다.

| 상태 | 사용자에게 알려야 하는 것 | 화면 원칙 |
| --- | --- | --- |
| 로딩 | 데이터를 확인 중임 | 이전 값을 새 값처럼 표시하지 않고 레이아웃 점프를 줄인다. |
| 빈 상태 | 무엇이 없고 어떻게 시작하는지 | 설명과 대표 시작 행동 하나를 제공한다. |
| `PENDING` | 행동은 끝났지만 금융데이터 검증을 기다림 | XP 완료와 금융 성장 대기를 분리한다. |
| `STALE` | 데이터가 오래되어 진행을 반영할 수 없음 | 마지막 동기화 시각과 복구 행동 하나를 제공한다. |
| `INSUFFICIENT` | 계산 데이터가 부족함 | 부족한 데이터와 가능한 행동형 대안을 설명한다. |
| 목표 필요 | 탐색은 가능하지만 명령 기능이 잠김 | 잠긴 이유와 목표 설정 CTA를 제공한다. |
| 오류 | 무엇이 실패했고 다시 할 수 있는지 | 원인과 재시도를 제공하고 입력 초안을 보존한다. |
| 완료 | 행동 보상과 금융 성장 중 무엇이 반영됐는지 | 두 결과를 시각적으로 분리한다. |
| 루틴 교체 | 현재 루틴과 새 루틴의 차이 | 취소하면 기존 루틴을 유지한다. |

## 7. 데이터·금융 안전 원칙

### 데이터 상태

- `FRESH`: 현재 계산과 진행 반영이 가능하다.
- `PENDING`: 새로운 금융데이터 반영을 기다린다.
- `STALE`: 데이터가 오래되어 새로운 진행 반영을 멈춘다.
- `INSUFFICIENT`: 계산에 필요한 데이터가 부족하다.

모든 계산 결과는 계산 버전, 데이터 상태, 마지막 동기화 시각과 함께 제공한다.

### 메이트 공개 범위

공개 가능: 익명 생활 맥락, 루틴 유형·유지 기간, 범위화된 행동 정보, 데이터 기준일.

공개 금지: 실명, 계좌번호, 정확 잔액, 거래 원문, 직장, 상세 위치, 원천 사용자 ID,
보유 종목과 상품 가입 이력.

### 코치·계산 책임

- MVP는 생성형 AI를 호출하지 않고 승인된 문구와 규칙 기반 코치만 사용한다.
- 목표값, 스탯, 진행률, 대표 활동은 코드가 계산한다.
- 화면은 계산을 재구현하지 않고 계약의 결과와 근거를 표시한다.
- 코치는 계산 결과를 설명하고 다음 허용 행동만 추천한다.

### 게임화 가드레일

- 공개 순위, 사용자 간 기여도 경쟁, 실패 손실을 제공하지 않는다.
- 투자금액·수익률·매매량·상품 가입에 XP나 보상을 지급하지 않는다.
- 포인트는 환금되지 않으며 MVP에서 외부 쿠폰으로 교환하지 않는다.
- 상품 정보 열람은 레이드·스탯·퀘스트 완료에 영향을 주지 않는다.

## 8. 팀이 혼동하면 안 되는 제품 규칙

- 목표는 기준선 진단 후 사용자가 하나를 확정하며, `일단 탐색하기`도 허용한다.
- 목표가 없으면 메이트 읽기만 가능하고 레이드·퀘스트 수락·루틴 적용은 잠긴다.
- 메이트에서 가져오는 것은 주 목표가 아니라 실행 루틴과 서브퀘스트다.
- 활성 루틴은 하나이며 교체에는 사용자 확인이 필요하다.
- 첫 루틴 적용 화면은 추천안 하나를 우선하고 세 강도 비교는 선택 기능이다.
- 퀘스트는 XP를 줄 수 있지만 금융 스탯과 레이드를 직접 변경하지 않는다.
- 금융 스탯과 레이드는 검증된 합성 금융데이터 재계산으로만 변한다.
- 투자 판단은 학습·점검 행동만 다루고 수익률·종목·매매를 다루지 않는다.
- 친구와 비교 탐색은 MVP에서 합성·읽기 중심이며 핵심 쓰기 흐름은 메이트 찾기와 루틴 적용이다.
- 하나 상품은 검수 카탈로그 정보이며 메이트·루틴 데이터와 분리한다.
- 상품 가입과 열람 보상은 제공하지 않는다.
- 기록은 `월간 발판 → 일일 바텀시트`이며 과거의 로드맵 탭 명칭을 사용하지 않는다.
- 시연 시간 진행은 합성 사용자와 시연 환경에서만 가능하다.

## 9. MVP 수용 기준과 검증

### 대표 수용 기준

- 사용자는 온보딩 후 목표를 확정하거나 탐색 모드로 진입할 수 있다.
- 탐색 모드에서 읽기는 가능하지만 목표가 필요한 명령은 일관되게 차단된다.
- 목표 확정 후 홈 레이드와 퀘스트 수락이 열린다.
- 곰·물개·토끼·새 리포트의 값과 설명 근거가 서로 분리된다.
- 사용자는 친구→메이트 찾기→비교 탐색의 차이를 이해할 수 있다.
- 핵심 흐름은 그룹→모험가→리포트→추천 루틴→적용 순서를 유지한다.
- 추천 루틴에는 추천안 하나와 선택 가능한 세 강도가 존재한다.
- 루틴 적용은 주 목표를 변경하지 않는다.
- 상품 정보 열람은 XP·금융 스탯·레이드를 변경하지 않는다.
- 퀘스트 수락, 행동 완료, 금융데이터 반영 상태가 구분된다.
- 여정 발판과 일일 바텀시트의 활동·금액·날짜가 일치한다.
- 여섯 번의 50만원 합성 저축 이후에만 200만원에서 500만원으로 진행한다.

### 필수 검증 시나리오

- 가입·로그인·세션 복원·로그아웃
- 온보딩 중단·재개, 목표 확정, 일단 탐색하기
- 목표 없는 레이드·퀘스트 수락·루틴 적용·개인화 상품 정보 차단
- 네 캐릭터 리포트의 정상·오래됨·데이터 부족 상태
- 퀘스트 수락, 행동 완료, 금융증거 대기, 데이터 반영 완료
- 친구 합성 읽기, 그룹 표본 부족, 모험가 공개 철회
- 추천 루틴 최초 적용과 기존 루틴 교체 취소·확정
- 하나 상품 정보 열람 전후 데이터 불변
- 발판 날짜 순서, 대표 활동 계산, 일일 상세 합계
- 7월→1월 여섯 번의 합성 저축과 목표 완료
- 모바일 360px·390px·430px의 잘림·겹침·스크롤

### 제품 지표

- 온보딩 완료율과 목표 확정/탐색 선택 비율
- 추천 모험가 카드→리포트 전환율
- 추천 루틴 확인→적용률
- 첫 퀘스트 수락·실행률
- 금융데이터로 검증된 첫 행동 완료율
- 4주 루틴 유지율
- 목표 필요 차단 후 목표 확정 전환율
- 데이터 불일치 신고율
- 투자 유도·개인정보 노출·상품 열람 성장 반영 `0건`

## 10. 업무 단계별 진행 기준

1. **제품 기준 확정**
   - 산출물: 팀 계획서, IA, 용어, MVP·제외 범위, 결정 로그.
   - 완료 조건: 목표·루틴·퀘스트·레이드·기록의 관계에 이견이 없다.
2. **계약 확정**
   - 산출물: 상태 모델, 계산 정책, OpenAPI, 예시 응답, 계약 검증.
   - 완료 조건: 화면과 예시만으로 대표 흐름·예외 흐름을 끝까지 따라갈 수 있다.
3. **디자인 인수·감사**
   - 산출물: 원본 동결, 화면·에셋 목록, `KEEP·REFINE·REBUILD·DROP` 분류.
   - 완료 조건: 누락 화면과 공통 상태가 식별된다.
4. **화면 구조 통합**
   - 산출물: 공통 셸, 내비게이션, 정상·빈·오류·데이터 상태.
   - 완료 조건: IA의 모든 MVP 화면을 모바일에서 탐색할 수 있다.
5. **제품 동작 연결**
   - 산출물: 목표, 레이드, 메이트 루틴, 퀘스트, 기록의 연결.
   - 완료 조건: 루틴 적용이 목표를 바꾸지 않고 금융데이터 반영 전후가 구분된다.
6. **통합 검증**
   - 산출물: 계약·E2E·모바일 검수·금지 요소 검사 결과.
   - 완료 조건: 합성 fixture와 제품 계약이 같은 흐름을 설명한다.
7. **시연·발표 준비**
   - 산출물: 결정적 90초 시나리오, 발표자료, 주장·근거표.
   - 완료 조건: 구현, 시연 전용 연출, 후속 기능이 명확히 구분된다.

- 이 문서는 팀의 읽기 시작점이며 상세 규칙을 임의로 복제해 별도 진실을 만들지 않는다.
- 제품 정책 변경은 관련 문서, OpenAPI 예시, 검증과 화면 인벤토리를 함께 갱신한다.
- 잠긴 결정을 바꾸면 결정 로그에 대체 이유와 영향 범위를 남긴다.
- Notion에는 원문 복제보다 문서 링크, 담당 상태와 회의 결정을 기록한다.
- 발표자료에서 새로운 정책을 정하지 않고 확정 문서를 요약한다.

## Assumptions

- 첫 플랫폼은 모바일 웹·PWA다.
- MVP 금융데이터는 모두 합성 데이터다.
- 유럽여행경비는 대표 시연값이며 유일한 목표 유형이 아니다.
- 친구와 비교 탐색은 합성·읽기 중심이고, 핵심 쓰기 흐름은 메이트 찾기와 루틴 적용이다.
- 실제 마이데이터, 실제 금융상품 가입, 생성형 AI, 실제 투자, 외부 보상은 MVP 이후 별도 검토한다.
