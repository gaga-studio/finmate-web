# 핀메이트 · 마이데이터 기반 방치형 금융 RPG 기획서

> 소셜 금융 앱에서 캐릭터 성장형 금융 습관 앱으로의 재설계
> 

---

# 0. 서비스 한 줄 정의

**핀메이트는 사용자의 소비·저축·투자 데이터가 캐릭터 성장과 전투 장면으로 보이고, 메이트와 비교하며 퀘스트를 수행하고, 로드맵으로 금융 흐름을 따라가는 마이데이터 기반 금융 습관 앱이다.**

발표용 한 줄 정의는 다음과 같다.

> **핀메이트는 마이데이터를 기반으로 금융 생활을 자동 반영하고, 이를 RPG 전투·성장 구조로 시각화한 금융 습관 앱이다.**
> 

---

# 1. 기획 방향

기존 핀메이트는 친구와 또래의 금융 행동을 비교하고, AI 코치와 미션을 통해 금융 습관을 형성하는 **소셜 금융 앱**이었다.

하지만 숫자 중심의 금융 UI는 20대 금융 초보자에게 어렵고, 비교만으로는 지속적인 행동 유도가 약할 수 있다.

이를 개선하기 위해 핀메이트는 기존 비교 구조를 유지하되, 전체 경험을 **RPG 비주얼과 성장 구조**로 재해석한다.

핵심 변화는 다음과 같다.

- 금융 상태를 숫자 대신 캐릭터 성장으로 보여준다.
- 사용자의 실제 금융 행동은 마이데이터로 자동 반영된다.
- 홈에서는 내 금융 상태가 **보스 전투 장면**으로 나타난다.
- 메이트에서는 또래, 친구, 조건 기반 사용자를 비교한다.
- 퀘스트에서는 실제 금융 행동을 수행한다.
- 로드맵에서는 월간 금융 흐름을 **여정 형태의 기록 UI**로 확인한다.

즉, 핀메이트는

**“비교 중심 금융 앱”에서 “자동 성장과 행동 유도 중심 금융 RPG형 앱”으로 전환**된다.

---

# 2. 왜 방치형 RPG인가

금융은 행동도 어렵고, 보상도 늦다.

반면 소비는 행동도 쉽고, 보상도 빠르다.

이 비대칭 때문에 많은 사용자가 금융 행동을 시작하거나 지속하기 어렵다.

핀메이트는 방치형 RPG 구조를 통해 이 문제를 해결한다.

## 2-1. 자동 반영 구조

방치형 RPG에서는 사용자가 계속 조작하지 않아도 캐릭터가 움직이고 성장한다.

핀메이트에서도 사용자가 매번 입력하지 않아도, 마이데이터를 통해 실제 금융 생활이 자동 반영된다.

```
소비 데이터 → 지갑 HP 변화
저축 데이터 → 저축력 변화
투자 데이터 → 성장력 변화
퀘스트 수행 → XP / 포인트 / 보상 획득
```

## 2-2. 느린 금융 보상의 즉시 체감

저축, 투자, 자동이체, 이자, 배당은 실제 체감까지 시간이 걸린다.

핀메이트는 이 결과를 다음과 같이 즉시 체감하게 만든다.

- 자동저축 완료 → 저축 캐릭터 강화
- 투자 리포트 확인 → 투자 캐릭터 성장
- 카페비 목표 달성 → 소비 관리 캐릭터 기여도 상승
- 퀘스트 완료 → 전투 진행률 상승, 보상 상자 해금

결국 사용자는 금융 데이터를 읽는 것이 아니라,

**“내 금융 생활이 지금 어떤 전투를 만들고 있는지”**를 보게 된다.

# 3. 핵심 사용자 경험 구조

## 기존 구조

```
비교
→ 인식
→ 미션
→ 포인트
→ 리포트
```

## 개선 구조

```
홈에서 자동 전투 확인
→ 메이트에서 비교
→ 퀘스트 수행
→ 전투 기여도/보상 증가
→ 로드맵에서 금융 기록과 흐름 확인
```

## 최종 루프

```
접속
→ 보스 전투/자동 성장 확인
→ 필요한 퀘스트 수행
→ 메이트 비교로 자극 받기
→ 보상 획득
→ 로드맵에서 기록과 성장 흐름 확인
→ 다음 행동 결정
```

# 6. 홈

# 6-1. 홈의 역할

홈은 핀메이트의 핵심 화면이며, 단순 상태 대시보드가 아니라 **RPG 게임 메인 화면**이다.

이 화면에서 사용자는 자신의 금융 생활이 어떻게 캐릭터 전투로 반영되고 있는지를 본다.

핵심 목적은 다음과 같다.

- 내 금융 상태를 한눈에 이해
- 자동 성장 / 자동 전투 중인 상태 확인
- 소비 / 저축 / 투자 / 미션 각각이 현재 어떤 역할을 하고 있는지 확인
- 각 영역을 탭해 상세 보고서로 이동

---

# 6-2. 홈 화면의 핵심 구조

## 1) 상단 영역

- 사용자 프로필
- 레벨
- 포인트/재화
- 종합 성장도
- 알림

## 2) 보스 전투 메인 영역

홈의 가장 중요한 부분이다.

### 화면 구조

- 중앙 또는 우측에 **오늘의 보스**
- 좌측 또는 하단에 **4명의 주요 캐릭터**
    - 소비 캐릭터
    - 저축 캐릭터
    - 투자 캐릭터
    - 미션 캐릭터
- 이 4명이 하나의 보스를 향해 싸우고 있음
- 전투는 자동 진행 중인 느낌으로 보여줌

### 보스 의미

보스는 현재 사용자의 금융 생활에서 가장 중요한 문제나 관리 대상이다.

예시:

- 카페비 몬스터
- 외식비 보스
- 과소비 골렘
- 저축 방해 몬스터
- 분산 부족 몬스터

즉, **현재 가장 관리가 필요한 금융 이슈가 보스로 시각화**된다.

---

# 6-3. 홈의 캐릭터 구조

각 캐릭터는 금융 영역 하나를 상징한다.

| 캐릭터 | 담당 영역 | 역할 |
| --- | --- | --- |
| 소비 캐릭터 | 소비 | 지갑 HP 관리, 예산 방어 |
| 저축 캐릭터 | 저축 | 저축력 강화, 방어형 기여 |
| 투자 캐릭터 | 투자 | 성장력 상승, 공격형 기여 |
| 미션 캐릭터 | 퀘스트 | 진행률 상승, 보조/버프 역할 |

이 캐릭터들은 단순 장식이 아니라 **각 영역의 상태와 연결된 인터랙션 요소**다.

---

# 6-4. 홈의 인터랙션

### 캐릭터 클릭 시

각 캐릭터를 누르면 해당 영역의 **상세 보고서**로 이동한다.

예:

- 소비 캐릭터 클릭 → 소비 보고서
- 저축 캐릭터 클릭 → 저축 보고서
- 투자 캐릭터 클릭 → 투자 보고서
- 미션 캐릭터 클릭 → 퀘스트 진행 보고서

즉, 홈의 전투 장면은 단순 비주얼이 아니라 **분석 화면으로 들어가는 메인 허브 역할**을 한다.

---

# 6-5. 홈 문구 예시

전투는 보이되 문구는 쉽게 간다.

예시 문구:

- 카페비 목표를 지켜 진행률이 12% 올랐어요.
- 자동저축이 완료되어 저축력이 올랐어요.
- 투자 흐름이 좋아져 성장력이 상승했어요.
- 오늘의 퀘스트 2개를 완료해 보상 상자가 가까워졌어요.
- 외식비가 늘어 지갑 HP가 조금 줄었어요.

---

# 6-6. 홈에서 제공하는 요약 정보

- 오늘의 보스
- 보스 진행률 / 남은 HP
- 각 캐릭터 기여도
- 지갑 HP / 저축력 / 성장력 / 안정도
- 오늘의 퀘스트 진행률
- 보상 상자 진행률
- AI 코치 한 줄 조언

# 7. 메이트

메이트는 비교 기능 전체를 담당하는 탭이다.

구조는 유지한다.

```
메이트
├─ 또래 비교
├─ 친구
└─ 비교 탐색
```

또는 UI 탭:

```
[또래 비교] [친구] [비교 탐색]
```

## 메이트의 역할

- 나와 비슷한 또래 그룹과 비교
- 연락처 기반 친구와 비교
- 조건 기반으로 익명 사용자 탐색
- 비교 결과를 통해 행동 동기 부여

---

# 8. 퀘스트

퀘스트는 사용자가 실제 금융 행동을 실천하는 공간이다.

## 역할

- 오늘 할 금융 행동 확인
- 퀘스트 수행
- 진행률 확인
- XP, 포인트, 보상 획득

## 예시 퀘스트

- 카페비 5,000원 이하로 관리하기
- 오늘 저축 3,000원 달성하기
- 투자 포트폴리오 확인하기
- 금융 퀴즈 3문제 풀기
- 비상금 목표 확인하기

## 퀘스트 카테고리

- 소비 퀘스트
- 저축 퀘스트
- 투자 퀘스트
- 금융지식 퀘스트

# 9. 로드맵

# 9-1. 로드맵의 역할

로드맵은 기존 기록 기능을 확장한 탭이다.

단순한 거래 내역 나열이 아니라,

**이번 달 내 금융 생활이 어떤 흐름으로 이어졌는지 보여주는 “여정형 기록 화면”**이다.

즉, 로드맵은

**“내가 이번 달 돈을 어떻게 흘려보냈고, 어디서 성장했고, 어떤 행동을 했는지”**를 시간 순서대로 보여준다.

---

# 9-2. 로드맵의 UI 방향

로드맵은 첨부 이미지와 같이 다음 성격을 가진다.

## 시각 구조

- 월 단위 화면
- 상단에 월 요약
- 아래에 **길처럼 이어지는 여정 구조**
- 날짜별 노드가 순서대로 이어짐
- 각 노드에는 해당 날짜의 금융 행동/이벤트가 표시됨

즉, 전통적인 리스트가 아니라

**“30일 금융 여정”**을 시각적으로 따라가는 구조다.

---

# 9-3. 로드맵 핵심 요소

## 1) 상단 요약 영역

- 월 선택
- 이번 달 기록 수
- 월간 수입
- 월간 지출
- 월간 저축
- 필요 시 투자 흐름 추가 가능

예시:

- 수입 +2,800,000원
- 지출 -163,400원
- 저축 +100,000원

---

## 2) 여정형 일자 노드

각 날짜별로 하나의 노드가 배치된다.

노드에는 다음 정보가 들어간다.

- 날짜
- 핵심 이벤트
- 금액
- 관련 카테고리 태그
- 완료 여부
- 이동 버튼

예:

- 7일: 지출 -24,500원
- 8일: 지출 -8,900원
- 9일: 투자 +50,000원
- 10일: 저축 +100,000원
- 11일: 수입 +2,800,000원
- 12일: 오늘 예산 28,000원
- 13일: 퀘스트 예정

---

## 3) 카테고리 태그

각 노드에는 그날의 성격을 짧은 태그로 붙인다.

예:

- 지출
- 수입
- 저축
- 투자
- 퀘스트
- 예정

---

## 4) 현재 위치 강조

오늘 날짜 노드는 크게 강조된다.

사용자는 로드맵에서

**“내가 지금 이번 달 여정 중 어디쯤 와 있는지”**를 직관적으로 알 수 있다.

---

## 5) 미래 일정 / 잠금 노드

아직 오지 않은 날짜는 연하게 처리하거나 잠금된 노드로 보여줄 수 있다.

예:

- 내일 예산
- 예정된 퀘스트
- 잠금 보상
- 월말 보상

이렇게 하면 로드맵은 과거 기록만 아니라 **앞으로 남은 금융 행동과 목표까지 연결되는 구조**가 된다.

---

# 9-4. 로드맵의 기능적 의미

로드맵은 단순 기록이 아니라 다음 역할을 한다.

- 금융 흐름을 스토리처럼 보여줌
- 사용자의 월간 행동을 쉽게 회고하게 함
- 어떤 날에 어떤 금융 행동이 있었는지 직관적으로 파악하게 함
- 퀘스트와 소비/저축/투자 흐름을 한 화면에서 연결
- “기록”을 지루한 숫자 목록이 아니라 성장 여정으로 재해석

---

# 10. AI 코치

AI 코치는 각 탭 안에서 맥락적으로 등장한다.

## 홈

- 현재 전투 상황 해석
- 어떤 캐릭터가 약한지 조언

예:

- 외식비가 늘어 소비 캐릭터의 기여도가 낮아졌어요.
- 자동저축이 완료되어 저축 캐릭터가 안정적으로 성장하고 있어요.

## 메이트

- 비교 결과 해석
- 또래/친구 대비 강약점 설명

## 퀘스트

- 오늘 수행할 금융 행동 추천

## 로드맵

- 이번 달 흐름 해석
- 자주 반복되는 소비 패턴 분석
- 다음 주 목표 추천

# 11. 개인정보 보호 방향

메이트 탭 비교 기능에서는 정확한 금액을 직접 공개하지 않는다.

## 공개 가능한 정보

- 퀘스트 달성 여부
- 스탯 변화 여부
- 성장 방향
- 기여도
- 함께 도전 참여 여부

## 공개하지 않는 정보

- 정확한 자산 금액
- 정확한 계좌 잔액
- 정확한 월급
- 세부 소비 내역
- 정확한 투자 원금

즉, 비교는 유지하되,

**금액 자체보다 행동과 상태 중심으로 보여주는 구조**를 유지한다.

# 13. 발표용 핵심 멘트

> 개선된 핀메이트는 금융 상태를 단순 숫자로 보여주는 앱이 아니라, 사용자의 금융 생활을 RPG 전투와 성장 구조로 시각화한 금융 습관 앱입니다.
> 
> 
> 홈에서는 소비, 저축, 투자, 미션을 상징하는 캐릭터들이 하나의 보스를 상대로 자동 전투를 진행하고, 각 캐릭터를 클릭하면 해당 금융 영역의 상세 보고서를 확인할 수 있습니다.
> 
> 메이트에서는 또래 비교, 친구 비교, 비교 탐색을 통해 비교 기능을 유지하고, 퀘스트에서는 실제 금융 행동을 수행하게 만듭니다.
> 
> 마지막으로 로드맵에서는 이번 달 금융 흐름을 30일 여정 형태로 시각화해, 사용자가 자신의 소비, 수입, 저축, 투자, 퀘스트 기록을 길을 따라 자연스럽게 이해할 수 있도록 합니다.
>
